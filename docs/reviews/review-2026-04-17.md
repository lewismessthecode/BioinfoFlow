## Cross-Cutting Themes

### 1. Authorization gap runs end-to-end (Critical)

The same vulnerability surfaces at three layers, reinforcing the severity:

- **API layer** — `/events/stream` and all `/agent/*/approvals` endpoints accept a resource ID and return data without verifying the caller owns the resource (`api/v1/events.py:18`, `api/v1/agent.py:288`).
- **Repository layer** — `StatsRepository.get_run_counts_by_status / get_project_count / get_recent_runs`, `RunRepository.list / list_by_statuses`, and `ConversationRepository.list` all **accept `user_id` but never filter on it** (`repositories/stats_repo.py:23-55`, `run_repo.py:15-51`, `conversation_repo.py:14-64`). Callers pass `user.id` assuming it's honored.
- **Audit layer** — Approval resolution persists `resolved_by="user"` as a string literal instead of the real principal, erasing attribution for a security-sensitive action (`api/v1/agent.py:314`).

**Impact:** Anonymous or cross-user clients can subscribe to other projects' SSE streams (run logs, agent activity, approvals), list and resolve other users' approvals, and see cross-user dashboard counts.

**Suggested fix:** Enforce scoping at the repository level — every method that accepts `user_id` should join through `projects` / `conversations` and filter. Then add a route-level ownership check that maps the resource ID back to an owner before any read/mutation. Track down every `resolved_by` / `actor` string literal.

### 2. Risk-gate bypass in subagents (Critical)

`run_subagent` invokes the child `agent_loop` without passing `db_session` or `conversation_id` (`runtime/subagent.py:63-72`). Inside the loop, `_check_risk` early-returns when either is missing (`runtime/loop.py:92-94`), so **every ACT_HIGH tool call in a subagent skips the approval workflow** — including `execute_code`, `background_run`, and `file_write`. Parent sessions enforce approval; spawned subagents silently do not.

**Suggested fix:** Thread `db_session` and `conversation_id` through `run_subagent` to the child loop, matching the parent call site in `agent_service.py:330-331`.

---

### 3. Silent failures cascade through hook/error boundaries (Critical → High)

- **Scheduler completion hooks** run sequentially with no per-step try/except (`scheduler/hooks.py:38-77`). A `PermissionError` in `cleanup_run` suppresses the audit log, notifications, *and* batch completion — and the exception escapes the caller, leaking the scheduler slot.
- **Agent service** calls `background_manager.shutdown()` in `finally` without a None guard, even though the manager is `None` when `allow_workspace_tools=False` (`agent_service.py:334`). Every default-project agent call raises `AttributeError` at turn end.
- **ResourceMonitor** — `float(line)` on nvidia-smi output throws `ValueError` on non-numeric lines (`"N/A"`), the exception escapes `_sample_sync`, and the `_latest` snapshot goes stale for up to `sample_interval` seconds (`scheduler/monitor.py:108`).

**Suggested fix:** Wrap each independent side-effect in its own try/except-and-log. Guard optional dependencies before access. Validate subprocess output before parsing.

---

### 4. Async/race correctness (High)

- **Slot accounting race** — workers read `self._slots.available` *before* `_claim_lock` serialises the DB claim, then `acquire()` *after* (`scheduler/scheduler.py:340-346`). Two concurrent workers can each claim weight-fitting tasks against the same stale snapshot and over-subscribe the pool.
- **Approval polling never sees writes** — `wait_for_approval` polls via `repo.get()` which hits the SQLAlchemy identity-map cache (`approval_service.py:170-172`). Another session's commit is invisible; polling spins the full 300 s timeout.

**Suggested fix:** Move the slot admission check *inside* `_claim_lock` (or release the claim back to QUEUED on slot-miss). Replace `session.get` in the approval poll with an explicit SELECT, or `session.expire(approval)` each tick.

### 5. Path contract v2 — incomplete migration (Critical / High)

- **Migration 0019 drops `workspace_path` without backfilling** (`alembic/versions/0019_add_project_storage_fields.py:20-37` + `0020_path_contract_v2.py:20-28`). Existing rows whose files lived outside the new managed layout lose their persisted location — no mapping into `storage_mode=external` + `external_root_path`.
- **Schema/model drift** — `projects.workspace_id` has no foreign key in any migration, yet the ORM declares one with `ON DELETE CASCADE` (`alembic/versions/0017_workspace_team_auth.py:110-128` vs `models/project.py:21-26`). Referential integrity isn't actually enforced at the DB level.
- **Downgrades are unsafe** — `0020` and `0021` reinstate columns with `"."` / empty defaults rather than reconstructing original values, so a rollback silently corrupts data.
- **Cleanup path traversal** — `WorkDirCleaner._candidate_paths` does `workspace_path / "runs" / run_id` with no validation that `run_id` lacks `..` (`scheduler/cleanup.py:66-76`). A corrupted DB row could cause `shutil.rmtree` to target an arbitrary directory.

**Suggested fix:** Write a one-off data migration that classifies each project into `managed` / `external` based on the old `workspace_path`. Add the missing FK migration. Mark 0020/0021 as irreversible (or implement real reconstruction). Validate `run_id` is a pure UUID / hex before constructing paths; assert the resolved path stays inside `workspace_path/runs`.

### 6. Engine correctness (High)

- **WDL adapter** fires `EngineEventType.ERROR` on any stdout line containing the substring `"error"` (`engine/adapters/wdl.py:121-125`). miniwdl's stdout routinely contains phrases like `no error`, `stderr_file`, `error_correction_enabled`. Because ERROR is terminal, successful zero-exit runs get marked failed.
- **`docker_mount_path_mappings` is `@lru_cache(maxsize=1)`** (`engine/miniwdl_mounts.py:72-104`) — the mount list is snapshot once per process. A remount/reattach goes unnoticed, and subsequent WDL runs hand miniwdl stale bind-mount sources.
- **`TimeoutWatcher` bypasses retry** — timed-out runs flow through `scheduler.cancel` rather than `_handle_task_failure`, so the retry policy is silently ignored for timeouts (`scheduler/timeout.py:74-77`).
- **Conversation history replay drops tool turns** — `_load_conversation_history` only restores `MessageType.TEXT` messages (`agent_service.py:497-521`). Resumed sessions that ended mid-tool-use produce structurally invalid OpenAI-format payloads → 400 from the provider.

### 7. Tool safety & contract violations (High)

- **`ShellTool` blocklist bypass** — strips path prefixes and checks `BLOCKED_TOKENS`, but the list omits `python`, `python3`, `bash`, `mv`, `curl`, `wget`, `pip`, `nc` (`tools/shell_tool.py:134-142`). Any of those lets the agent escape the sandbox. Tool is classified `ACT_LOW` (no approval gate under `auto` policy) and uses `create_subprocess_shell` — full shell interpretation.
- **`_register_base_tool` mutates `get_schema()` output** (`runtime/dispatch.py:263`) — `v.pop("required", False)` on the returned dict. Any tool that caches/returns a shared schema dict permanently loses its `required` declarations after the first `build_dispatch_map` call.
- **RunService facade regression** — `list_runs`, `get_run`, `get_run_audit`, `delete_run`, `list_outputs`, `build_output_archive` now contain business logic and reach into subservice private members (`_require_run`, `_require_run_access`) (`services/run_service.py:120`). This violates the facade-only rule enforced during the 2026-04-04 prune and re-entangles the split.
- **Envelope inconsistency** — agent rate-limit branch returns a bespoke `JSONResponse` with `{success, error}` instead of the full envelope (`api/v1/agent.py:66`).

**Suggested fix:** Switch `ShellTool` to an explicit allowlist or promote to `ACT_HIGH`. Deep-copy schema entries before mutation. Move RunService helpers back into the delegate services as public methods. Always return the full envelope with `Retry-After` as a header.

---

### 8. Frontend i18n and accessibility parity (High)

- **`KeyboardShortcutsOverlay`** never imports `useTranslations` — every visible label hardcoded (`components/bioinfoflow/chat/keyboard-shortcuts-overlay.tsx:11-55`). Also lacks `role="dialog"`, `aria-modal`, focus trap, and Escape-restore-focus.
- **`ChatErrorBoundary`** fallback UI strings are hardcoded English (`chat/chat-error-boundary.tsx:68-88`).
- **`FileBrowserDialog`** mixes `useTranslations("common")` with raw English error and empty-state strings (`file-browser-dialog.tsx:81-319`).
- **Root `metadata` export** is a static English object ignoring locale (`app/layout.tsx:19-41`); Chinese users see English browser titles / descriptions.
- **`ConnectionStatus` tooltip** wraps a plain `div` via Radix `asChild`, making it unfocusable and invisible to screen readers when connected.

**Suggested fix:** Swap custom overlays for existing `Dialog` primitives. Convert the root layout `metadata` export into `generateMetadata` async. Add the missing keys to both `messages/en.json` *and* `messages/zh-CN.json`. Consider adding an ESLint rule (`react/jsx-no-literals` with a whitelist) or a pre-commit grep to flag unwrapped string literals inside JSX.

## Recommended Remediation Order

1. **Authorization (day 1–3)** — close the repo-level scoping holes first (stats/run/conversation), then add route-level ownership checks on `/events/stream` and approval endpoints, fix `resolved_by` attribution. Security priority.
2. **Subagent approval bypass (day 1)** — one-line fix in `run_subagent`; blocks ACT_HIGH escape.
3. **Hook failure isolation + `background_manager` None guard (day 1)** — low-effort, high-impact reliability.
4. **Path contract v2 cleanup (day 2–4)** — write the backfill migration for `workspace_path → storage_mode`, add the `workspace_id` FK migration, mark 0020/0021 irreversible, harden `run_id` path assembly.
5. **WDL error detection + miniwdl mount cache (day 2)** — replace substring match with structured/exit-code signal; replace `lru_cache` with TTL or startup snapshot documented behavior.
6. **Slot-accounting race + approval polling cache (day 3)** — move slot check inside `_claim_lock`; use `session.expire` or explicit SELECT in polling.
7. **Shell tool hardening (day 3)** — allowlist or promote to `ACT_HIGH`.
8. **RunService facade re-split (day 4)** — move leaked methods back to delegate services.
9. **Frontend i18n & a11y sweep (day 4–5)** — extract hardcoded strings, swap custom overlay for `Dialog`, convert root `metadata` to `generateMetadata`. Add ESLint `no-literals` rule to prevent regression.
10. **Timeout → retry routing, resource-monitor sample hardening, envelope consistency (day 5)** — smaller correctness fixes.
